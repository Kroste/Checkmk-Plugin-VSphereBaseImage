**Roadmap 1c ist drin**: nach Snapshot hebt das Plugin das Master-Image des
zugehörigen Citrix Machine Catalogs automatisch auf den neuen Snapshot. Damit
ist der Baseimage-Update-Workflow End-to-End erledigt — Agent-Update →
vSphere-Snapshot → Citrix-Katalog-Publish, ohne Handgriff im DDC.

## Neu

**Citrix-CVAD-On-Prem-Machine-Catalog-Update**

Nach dem Snapshot ruft der `BatchRunner` das `CitrixClient.PublishMasterImageAsync`
gegen die Citrix-Orchestration-REST-API (`POST /citrix/orchestration/api/…`):

- Auth via `POST tokens` mit Basic (Domänen-User aus dem seit v0.3.0
  konfigurierbaren DDC-Anmeldedialog) → Bearer, alle weiteren Requests mit dem
  Token im `Authorization: Bearer …`-Header. 401 löst einen einmaligen
  Token-Refresh + Retry aus.
- Site-ID kommt via `GET Sites` — die erste Site gewinnt, cached für den Batch.
  Sollte für die Fachbereichs-Single-Site-Umgebung reichen; für Multi-Site
  wäre eine Auswahl in den Settings der nächste Schritt.
- Master-Image-Pfad wird aus dem aktuellen Katalog-Pfad abgeleitet:
  `XdHyp:\HostingUnits\{unit}\{oldVm}.vm\{oldSnap}.snapshot` → `\{newVm}.vm\{newSnap}.snapshot`.
  Damit muss die **Hosting-Unit nicht separat konfiguriert** werden — sie steckt
  bereits in der Katalog-Metadata.
- Auslöser: `POST Sites/{sid}/MachineCatalogs/{id}/$UpdateProvisioningScheme`
  mit `{ MasterImage: <neuer Pfad> }`.
- **Non-fatal**: Publish-Fehler bricht die restliche VM-Bearbeitung nicht ab.
  Update + Snapshot sind ja schon durch — der Admin kann im DDC manuell
  nachziehen, das wird pro VM im Log summiert.

**Katalog-Picker vor dem Batch**

Neuer Dialog `CatalogPickerDialog` blendet sich zwischen „Batch starten" und
CredentialDialog ein, mit einer Zeile pro sichtbarer VM:

- ComboBox mit allen Machine Catalogs vom DDC (`GET Sites/{sid}/MachineCatalogs`).
- **Automatischer Vorschlag** über die längste gemeinsame Namens-Präfix-Match
  (case-insensitive, min. 3 Zeichen): `CTX01100` → Katalog `CTX01`; „Vorschlag"-Label
  markiert die Vorbelegung als überschreibbar.
- Leerlassen = kein Publish für diese VM (nur Update + Snapshot).
- DDC nicht erreichbar / keine Anmeldedaten → leere Katalog-Liste; der Batch
  läuft trotzdem, nur eben ohne Publish-Optionen im Picker.

## Neuer Batch-Ablauf pro VM

```
Katalog-Picker (VM ↔ Machine Catalog, Vorschlag via Praefix-Match)
    ↓
Ausgangs-Power-State merken
    ↓
(falls aus) PowerOn
    ↓
warten auf VMware-Tools
    ↓
warten auf Ping
    ↓
Agent-Update (IAgentUpdater aus Plugin 1)
    ↓
Guest-Shutdown
    ↓
warten auf Power-Off
    ↓
Snapshot anlegen  (checkmk-update-YYYYMMDD-HHmmss)
    ↓
(wenn Katalog zugeordnet) Citrix-Publish  ← NEU
    ↓
(wenn wasOn) PowerOn → VM in Ausgangszustand
```
