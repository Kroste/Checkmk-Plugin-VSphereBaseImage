Erstes Release des vSphere-Baseimage-Plugins fürs Checkmk Cockpit.

Löst das CTX-Baseimage-Problem: die Baseimages sind normalerweise aus, werden
nachts neu ausgerollt, jede tagsübliche Änderung geht verloren. Dieses Plugin
fährt alle Baseimages im Batch hoch, aktualisiert den Checkmk-Agent (über das
`Checkmk-Plugin-AgentUpdater`), fährt sie wieder herunter.

## Was drin ist

- **Eigener Tab „vSphere Baseimages"** neben Status/Hosts/Dashboard (via
  `ITabContribution`, Order 1000).
- **vCenter-REST-Client** (v7+ Session-Auth, Basic → Token, auto-relogin bei 401).
- **VM-Grid** mit Filter (Regex oder Include-Liste, analog Cockpit-Host-Filter
  aber eigener Typ `VmFilter` — VMs sind keine Hosts).
- **Batch-Workflow** pro VM: Power-On → warten auf VMware-Tools → warten auf
  Ping → Agent-Update über `IAgentUpdater` aus Plugin 1 → Guest-Shutdown →
  warten auf Power-Off. Fehler brechen die Batch nicht ab.
- **DPAPI-verschlüsselter Credential-Store** für vCenter (keine Vorbelegung mit
  Windows-User — im Fachbereich weicht der vCenter-User ab).
- **Admin-Credentials für die Remote-PowerShell** werden pro Batch-Lauf im
  Dialog abgefragt, nicht persistiert.

## Voraussetzungen

- Windows.
- Checkmk Cockpit **≥ v1.7.0**.
- [`Checkmk-Plugin-AgentUpdater`](https://github.com/Kroste/Checkmk-Plugin-AgentUpdater)
  im gleichen `plugins/`-Ordner.
- vCenter-Konto + Admin-Konto auf den Baseimage-VMs (WinRM aktiv).

## Bewusste Grenzen v0.1.0

- **Kein GUI-Filter-Manager**: Klick auf „Filter verwalten…" öffnet die
  `filter.json` im OS-Editor. GUI kommt in v0.2.
- **Sequenzieller Workflow**: eine VM nach der anderen. Parallel wäre möglich,
  aber die Ping/WinRM-Timeouts vertragen sich schlecht mit Deploy-Storms.
- **Timeout-Konstanten hart codiert**: PowerOn/Tools = 10 min, Shutdown = 5 min.
  Wenn's bei euch nicht reicht → Issue.

## Installation

ZIP herunterladen, `CheckmkPlugin.VSphereBaseImage.dll` in `plugins/` neben
`Checkmk.App.exe` legen, Cockpit starten. Details siehe
[README](https://github.com/Kroste/Checkmk-Plugin-VSphereBaseImage/blob/v0.1.0/README.md).
