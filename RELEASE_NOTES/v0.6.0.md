**Kontextmenü am VM-Grid** — analog zum Cockpit. Die zwei täglichen
Handgriffe (RDP zum Draufschauen, Ping zum Netz-Check) sind jetzt direkt
an der Zeile, ohne Umweg über den vSphere-Web-Client oder ein zweites
Fenster.

## Neu

**Rechtsklick auf eine VM-Zeile:**

- **RDP öffnen** — `mstsc /v:<vm-name>` startet die Windows-RDP-Session
  gegen die VM. Baseimages tragen den Windows-Hostnamen als VM-Namen, also
  reicht der VM-Name als Ziel. Sollte das im Fachbereich mal auseinander-
  laufen (VM-Name ≠ Hostname), kommt ein Domain-Suffix als Plugin-Setting.
- **Ping (Fenster)** — `cmd /k ping -t <vm-name>` in einer neuen Konsole
  für den kontinuierlichen Reachability-Check (z. B. wenn der Batch beim
  Ping-Warteschritt hängen bleibt).
- **VM-Name kopieren** / **VM-ID kopieren** — der Name geht in
  PowerShell-Skripte oder Ticket-Referenzen, die ID (`vm-1234`) ist für
  vCenter-REST/PowerCLI-Debug hilfreich.

**Doppelklick** auf eine VM öffnet direkt RDP — bei Windows-Baseimages
ist das der Regelfall, deshalb als Shortcut zusätzlich zum Kontextmenü.
