**Bugfix**: das Plugin meldete an das Cockpit die Version `0.0.0`, auch nach
Update. Selbes Problem wie im AgentUpdater-Plugin.

## Behoben

`Directory.Build.props` hatte keinen `<MinVerTagPrefix>v</MinVerTagPrefix>`.
Die Tags heißen `v0.2.0`, `v0.2.1`, … — MinVer mit leerem Default-Prefix
hat sie ignoriert und stattdessen die Default-Version `0.0.0-alpha.0` +
Commit-Height gemeldet. Das `Plugin.cs`-Fallback (`?? "0.0.0"`) griff.

Fix analog Cockpit: `<MinVerTagPrefix>v</MinVerTagPrefix>` gesetzt. Ab
v0.2.4 stimmt die Versionsanzeige.
